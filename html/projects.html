geo-py library

Languages/libraries: Python, Jupyter Notebook

geo-py is a collection of Python classes representing design charts and/or design processes from the Naval Facilities Engineering Command geotechnical manuals, automating tasks that would otherwise yield subjective results or take a long time to do by hand. The most exciting piece of this project is the deep foundations design implementation. With some notable exceptions, the Drilled_Shaft_Design class can analyze a wide range of types and sizes of deep foundations and present the results immediately, allowing for tweaking of design parameters with no additional work and without introducing human calculation errors. Like any model, the output is only as good as the input, so it is imperative that the user deeply understands the design process and the input parameters used to obtain the results. The project is usable as a collection of classes that can be imported into another script or used in the terminal, and is also available as a Jupyter Notebook.

QC Proposal Builder

Languages/libraries: Python, docx, openpyxl

My QC Proposal Builder is an example of a practical tool that I use in my engineering practice. The end result is a faster and less error-prone method of creating routine documents at work. The build_qc_proposal.py script pulls data from the config.py file to create a proposal for quality control engineering services tailored to a specific job, built using standardized text blocks. The user modifies the client and project information in the config.py file and builds the budget using the estimate_sheet.xlsx file. The build_qc_proposal.py script uses the qc_propsal_template.docx file as a starting point containing the company logo and page formatting, adding in the standard text blocks, budget information, and unique client and project information to build the document and send to a new output file.
